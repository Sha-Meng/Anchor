# Game Jam Planner

## 使用原则

目标是尽快得到可玩的最小闭环，而不是完整商业方案。先收敛玩家体验，再决定系统数量。

## 工作流

1. 明确 jam 约束：主题、时长、目标平台、人数、已有资产、必须展示的亮点。
2. 提炼一句话玩法：玩家是谁、做什么、为什么紧张或有趣。
3. 定义核心循环：输入、决策、反馈、奖励或失败。
4. 划分范围：
   - 必做：没有它就不能玩。
   - 应做：显著提升体验但不阻断提交。
   - 可砍：延期不会破坏核心循环。
5. 输出 MVP 验收：3 到 5 条可试玩判断，不写抽象愿景。

## 输出模板

```markdown
## 一句话玩法

## 核心循环

## MVP 范围
- 必做：
- 应做：
- 可砍：

## 首个可玩版本验收

## 最大风险
```

## 约束

- 默认使用 Tuanjie Editor 1.9.3 / Unity 2022.3 思路。
- 不建议在 jam 中引入重架构、复杂联网、持久化经济系统或大型内容管线，除非用户明确要求。
- 如果方案超过 2 到 3 个核心机制，主动建议砍到 1 个主机制加 1 个变化。
