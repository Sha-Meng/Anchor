# Jam Test Acceptance

## 使用原则

验收服务于提交，不追求完整 QA。优先发现会阻止试玩、理解或提交的缺陷。

## 验收层级

1. 冒烟测试：项目能打开、无阻断编译错误、主场景能进入、能完成一局或核心循环。
2. 玩法验收：玩家输入有效、失败/胜利条件明确、重试路径可用。
3. 反馈验收：关键操作有视觉/音频/数值反馈，错误操作有可理解反馈。
4. 稳定性验收：重复 3 次核心循环无明显卡死、异常刷屏或状态残留。
5. 提交验收：构建目标、场景列表、说明文本、截图或录屏材料齐全。

```markdown
## 手工验收

```

## Unity/Tuanjie 验证

- 如 MCP 可用，优先读取 Console、运行测试、获取场景信息或截图。
- 如 MCP 不可用，给出 Editor 内手工步骤。
- 自动测试只覆盖稳定规则；手感、节奏和可读性必须保留人工试玩判断。
