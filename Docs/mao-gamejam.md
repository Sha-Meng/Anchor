# 锚 GameJam   
# 开发引擎   
Unity   
# 平台   
移动端   
# 背景设定（不重要 可以忽略）：   
两名登山者被第三者队友陷害跌落山谷 他们只有一根20米的登山绳和五根登山锚钉子；   
面对一眼望不到头的崖壁，他们只能选择……   
# 玩法：   
双人协作登山   
# 核心机制：   
利用锚钉和绳索，和你的队友一起安全的登上山顶   
锚钉：绳子可以从其中穿过  当体力不支 或没吸附到正确的攀登点上时，锚钉会保护玩家减少其所受伤害；游戏全局总共五个锚钉，前方的玩家使用锚钉后，后方的玩家需要将锚钉回收。然后交替领头（锚钉无法从玩家之间互换）。铆钉只分为玩家携带和场上已部署两类核心状态；玩家靠近已部署铆钉且自身处于 stable 状态时，会出现回收按钮，点击即可直接回收。若回收的铆钉正在承担保护作用，绳索会立即按剩余铆钉重算，可能导致队友坠落距离和伤害上升。双人联机时，成功打钉和成功回收都需要作为事件同步给另一名玩家，保证双方看到一致的铆钉、库存和绳路变化。   
绳子：绳子是和锚钉配合使用的安全装置，默认总长为20米；每次打铆钉不会增加绳长，而是让绳子多一个穿过点/转折点。当玩家不慎跌落时，会把整段绳子拉直，起到缓冲和保护作用；玩家跌落距离约等于绳子总长度-最上方的锚钉与下方玩家之间的已占用距离。多余的绳长表现为绳段下垂、下方玩家身边的绳圈或 UI 松弛量提示；下方玩家收绳时，绳圈/松弛量减少。   
主动收绳救援：为了强化双人协作，当上方玩家坠落时，下方玩家会进入短暂救援窗口。下方玩家可以快速点击“收绳”来收紧绳索，减少本次坠落的自由下坠距离或伤害倍率；收绳效果有上限，不能完全抵消错误。若下方玩家自身处于脱力、坠落、受击或不可交互状态，则收绳无效。   
属性：   
1.耐力：攀爬需要消耗耐力值 两只手分别消耗耐力，但一只手处于摸索状态时，另一只手的耐力会加速消耗（表现上会发生轻微抖动效果）   
2.生命：每次掉落会消耗生命值，伤害与掉落距离正相关；锚钉保护、双方保持合理距离，以及下方玩家及时收绳都能降低伤害   
可攀附点：每只手臂同一时刻最多覆盖三个攀附点（过密会影响摸索体验）   
可攀附点范围：

抓点类型：
1.有效抓点：可以被手部抓住并产生受力。每个有效抓点有一个稳定值，范围建议为0到1，数值越高越容易稳定攀附。
2.障碍点（Obstacle）：不是抓点，而是危险或干扰区域，例如尖刺、碎石、流沙凸起、不可抓边缘。手部碰到障碍点时，该手不产生有效受力，并交给角色表现或生命系统处理受击、扣血或反馈。
3.假点（Fake）：看起来像有效抓点，摸索时可以先给短暂震动或提示，但被识别后会失效。假点失效后稳定值视为0，该手不产生有效受力，用于制造误判和风险。
4.无效区域：没有任何可抓事实，不产生受力，也不提供抓点反馈。

抓点稳定值：
稳定值表示单个抓点的抓附质量，不直接等同于角色整体稳定状态。关卡系统只负责提供点位类型和稳定值，受力系统结合左右手、耐力和身体状态判断角色最终是否稳定。

建议规则：
- 稳定值=0：完全无效，不能受力。
- 稳定值低于最小抓附阈值：判定为低质量抓点，默认不能产生有效受力。
- 稳定值达到最小抓附阈值但低于稳定阈值：可以短暂受力，但更容易抖动、加速消耗耐力或缩短脱力容忍时间。
- 稳定值达到稳定阈值：正常有效受力。
- 障碍点和已暴露假点优先级高于稳定值，即使数值配置不为0，也应被受力系统视为无效受力。
# 系统分工：
整个游戏的核心玩法按职责拆成几个相互解耦的部分，方便并行开发和后续调参。

1. 角色3C系统：
负责控制整个攀爬过程中的角色操作、镜头、手感和表现。玩家主要操控布娃娃的左右手，系统需要处理手部移动、吸附、摸索、耐力表现、脱力表现、坠落表现和玩家反馈。

2. 关卡系统：
负责关卡初始化和关卡数据管理，包括出生点、终点、可攀附点、障碍物、假点、锚点位置和关卡难度配置。关卡系统只提供场景事实，不直接决定角色是否脱力或受伤。关卡系统需要向受力系统或其适配器提供手部位置附近的抓点查询结果，包括点位类型、稳定值、假点是否已暴露、点位标识和表面方向等信息。

3. 受力系统：
负责判定当前角色所处姿态是否是合法受力姿态。它根据左右手接触、抓点类型、抓点稳定值、假点暴露状态、耐力和身体状态输出判定结果，例如稳定、左手脱力、右手脱力、双手脱力、坠落等。受力系统不直接查询场景，也不直接移动角色；它消费适配器提交的轻量输入快照，并输出可被角色3C和铆钉绳索系统消费的结果。

受力系统接入边界：
- 角色3C系统提供左右手位置、接触状态、当前是否抓住候选点，以及身体是否已经处于坠落或受击等状态。
- 关卡系统提供给定手部位置附近的抓点事实，不返回最终稳定结论。
- 耐力系统提供左右手当前耐力和是否低于可受力阈值。
- 受力适配器负责把角色3C、关卡和耐力信息组装成一帧输入快照。
- 受力评估器只做轻量规则计算，支持高频调用和单元测试；在其他系统未完成时，可以用手写输入快照完成单元测试。

4. 铆钉和绳索系统：
负责铆钉库存、插锚、拔锚、打钉/回收事件同步、绳长约束、坠落时的保护分析、主动收绳救援和伤害判定。当角色坠落时，该系统根据20米默认绳长、角色位置、最上方已部署锚钉、回收后重算的绳索路径和下方玩家的有效收绳量判断是否受到保护，并计算最终伤害输入。

# 操作体验：   
pc端：   
利用的左右摇杆的震动反馈，摸索着找到可以攀附或下锚的地方   
左摇杆控制左手，右摇杆控制右手   
移动遥感时，手会以一定加速度移动（数值可调）   
队友坠落时，下方玩家可以快速点击收绳键/按钮进行救援；有效点击会让绳索更紧、救援计量增加，并降低本次坠落伤害   
   
移动端：   
横屏操作，左手拇指滑动屏幕操作角色的左手   
右手拇指滑动屏幕操作角色的右手   
滑动过程中，通过手机的震动反馈，感受角色的手和可攀附点的距离   
屏幕中央下方有基础操作按钮   
插锚   
拔锚   
队友坠落时，屏幕中央下方额外高亮“收绳”按钮，下方玩家可快速点击收绳来减少队友伤害；无效点击需要给出冷却、不可交互或窗口结束反馈   
   
角色相关：   
角色采用布娃娃系统，玩家只能操控布娃娃的手部，身体其他部位不用控制   
对手部控制时 手部始终处于对崖壁平面（可能存在凹凸）呈吸附状态   
   
# 关卡设计：   
提供三个关卡 难度递进   
关卡1:教学关无任何障碍   
关卡2:存在一些固定障碍物，摸点时要注意避开，否则会扣除生命   
关卡3:存在一些假点，玩家触摸时会在短暂震动后停止震动   
   
# 场景设计   
崖壁存在一层流沙层
