# 锚 GameJam   
# 开发引擎   
Unity   
# 平台   
移动端   
# 背景设定（不重要 可以忽略）：   
两名登山者被第三者队友陷害跌落山谷 他们只有一根20米的登山绳和五根登山锚钉子；   
面对一眼望不到头的崖壁，他们只能选择……   
# 玩法：   
双人协作登山   
# 核心机制：   
利用锚钉和绳索，和你的队友一起安全的登上山顶   
锚钉：绳子可以从其中穿过  当体力不支 或没吸附到正确的攀登点上时，锚钉会保护玩家减少其所受伤害；游戏全局总共五个锚钉，前方的玩家使用锚钉后，后方的玩家需要将锚钉回收。然后交替领头（锚钉无法从玩家之间互换）   
绳子：绳子是和锚钉配合使用的安全装置，当玩家不慎跌落时，会把整段绳子拉直，起到缓冲和保护作用；绳子长度相对固定，也就是说玩家跌落距离=绳子总长度-最上方的锚钉与下方玩家之间的距离   
属性：   
1.耐力：攀爬需要消耗耐力值 两只手分别消耗耐力，但一只手处于摸索状态时，另一只手的耐力会加速消耗（表现上会发生轻微抖动效果）   
2.生命：每次掉落会消耗生命值，生命值和掉落距离呈反比，鼓励双方玩家尽可能的离得近一些   
可攀附点：每只手臂同一时刻最多覆盖三个攀附点（过密会影响摸索体验）

# 系统分工：
整个游戏的核心玩法按职责拆成几个相互解耦的部分，方便并行开发和后续调参。

1. 角色3C系统：
负责控制整个攀爬过程中的角色操作、镜头、手感和表现。玩家主要操控布娃娃的左右手，系统需要处理手部移动、吸附、摸索、耐力表现、脱力表现、坠落表现和玩家反馈。

2. 关卡系统：
负责关卡初始化和关卡数据管理，包括出生点、终点、可攀附点、障碍物、假点、锚点位置和关卡难度配置。关卡系统只提供场景事实，不直接决定角色是否脱力或受伤。

3. 受力系统：
负责判定当前角色所处姿态是否是合法受力姿态。它根据左右手接触、可攀附点、障碍、假点、耐力和身体状态输出判定结果，例如稳定、左手脱力、右手脱力、双手脱力、坠落等。

4. 铆钉和绳索系统：
负责铆钉库存、插锚、拔锚、绳长约束、坠落时的受力分析和伤害判定。当角色坠落时，该系统根据绳长、角色位置和最上方有效锚钉判断是否受到保护，并计算最终伤害。

# 操作体验：   
pc端：   
利用的左右摇杆的震动反馈，摸索着找到可以攀附或下锚的地方   
左摇杆控制左手，右摇杆控制右手   
移动遥感时，手会以一定加速度移动（数值可调）   
   
移动端：   
横屏操作，左手拇指滑动屏幕操作角色的左手   
右手拇指滑动屏幕操作角色的右手   
滑动过程中，通过手机的震动反馈，感受角色的手和可攀附点的距离   
屏幕中央下方有两个按钮   
插锚   
拔锚   
   
角色相关：   
角色采用布娃娃系统，玩家只能操控布娃娃的手部，身体其他部位不用控制   
对手部控制时 手部始终处于对崖壁平面（可能存在凹凸）呈吸附状态   
   
# 关卡设计：   
提供三个关卡 难度递进   
关卡1:教学关无任何障碍   
关卡2:存在一些固定障碍物，摸点时要注意避开，否则会扣除生命   
关卡3:存在一些假点，玩家触摸时会在短暂震动后停止震动   
   
# 场景设计   
崖壁存在一层流沙层
