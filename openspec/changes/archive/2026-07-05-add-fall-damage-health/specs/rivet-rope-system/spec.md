## MODIFIED Requirements

### Requirement: 伤害结果计算
系统 MUST 根据坠落保护结果和主动收绳结果生成结构化伤害结果。伤害结果 MUST 至少包含坠落者 ID、是否受保护、保护铆钉 ID、第一个保护铆钉 ID、坠落者到第一个保护铆钉的绳索段长度、估算自由坠落距离、有效收绳量、救援减伤倍率、伤害倍率、建议伤害值和主要原因。对于相同保护状态和相同收绳效果，坠落者到第一个保护铆钉的绳索段长度越大，建议伤害值 MUST 不小于较短距离的伤害值。没有保护铆钉时，系统 MUST 输出无保护坠落原因，并使用无保护坠落配置计算建议伤害值。系统 MUST NOT 直接修改玩家生命值。

#### Scenario: 受保护坠落降低伤害
- **WHEN** 两次坠落的估算自由坠落距离相同，但一次存在已部署保护铆钉，另一次没有已部署保护铆钉
- **THEN** 受保护坠落的建议伤害值 MUST 小于或等于无保护坠落的建议伤害值

#### Scenario: 第一保护铆钉绳索段变长伤害不降低
- **WHEN** 同一保护状态下发生两次坠落，第二次坠落者到第一个保护铆钉的绳索段长度大于第一次
- **THEN** 第二次输出的建议伤害值 MUST 大于或等于第一次

#### Scenario: 有效收绳降低伤害
- **WHEN** 两次坠落的保护状态和第一个保护铆钉绳索段长度相同，但第二次存在更高的有效收绳量
- **THEN** 第二次输出的建议伤害值 MUST 小于或等于第一次

#### Scenario: 伤害结果包含第一个保护铆钉调试信息
- **WHEN** 系统输出受保护坠落伤害结果
- **THEN** 结果 MUST 包含第一个保护铆钉 ID、坠落者到该铆钉的绳索段长度和主要原因，用于调试距离伤害来源

#### Scenario: 无保护坠落不伪造保护铆钉
- **WHEN** 玩家坠落且当前绳索链路中不存在可保护该坠落者的已部署铆钉
- **THEN** 伤害结果 MUST 标记为无保护坠落，并 MUST NOT 填写虚假的第一个保护铆钉 ID

#### Scenario: 伤害结果不扣生命
- **WHEN** 系统输出坠落伤害结果
- **THEN** 系统 MUST 只返回结构化伤害结果，并 MUST NOT 直接修改玩家生命值
