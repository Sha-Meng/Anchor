## Why

Anchor 计划打造一款以"攀爬"为核心体验的手机游戏，第一优先级是把角色攀爬 3C（Character / Control / Camera）做扎实。核心玩法是**左手—右手交替**攀爬：屏幕左右各有输入区，只有"当前攀爬手"一侧生效，触点映射为该手的目标位置并驱动手部 IK 去寻找墙上的**铆钉抓力点**；越靠近铆钉，手机震动越强，抓到即换手，松手即放弃归位。

在初版 2D 方案基础上，本作**升级为 3D 表现**：墙体需要体现**凸起/浮雕**的立体感，铆钉是墙面上真实的凸起抓点；角色接入**布娃娃（ragdoll）系统**。新增**耐力条**机制：攀爬过程中耐力持续下降，抓住铆钉后快速恢复；耐力归零角色会**摔落一段距离**，摔落由布娃娃系统驱动。项目已内置 `FImpossible Creations / Ragdoll Animator 2`（3D 布娃娃 + 物理混合 + 冲击/起身工具），正好用于攀爬时动画驱动、摔落时切换全布娃娃。

当前仓库的 `Assets/ClimbGame` 只是 2D 方向摇杆式自由攀爬原型，与本 3C 模型不同；`Assets/DesignerSpace` 只是震动分档预研。因此需要本变更定义完整的 3D 攀爬 3C、手部 IK、铆钉与靠近度震动、放大镜观察窗、耐力与布娃娃摔落，以及可实时调参的配置体系。

## What Changes

- 攀爬场景升级为 **3D**：正对镜头的立体墙面，体现**凸起/浮雕**表现；铆钉为墙面上的立体凸起抓点。
- 角色从墙面中心起攀，**初始双手都在默认位置**；**由玩家先触发的一侧决定首个攀爬手**，之后左右手交替。
- 屏幕左右各一个**输入区**，仅"当前攀爬手"同侧生效；触点通过摄像机射线映射到 3D 墙面上，作为该手 IK 的末端目标，手随触点移动。
- 触点附近显示一个小圆形**放大镜**（用 **RenderTexture** 实现）放大观察当前手的着力点。
- 墙面分布**铆钉抓力点**；手越靠近铆钉震动越强，进入吸附半径即"完成本次攀爬"并**换手**；抓握前触点断开则**放弃**，该手平滑回默认位置且不换手。
- **躯干重心跟随双手中点**：抓到新铆钉后，身体朝当前两只手（锚定手 + 新抓点）的中点移动。
- 手部运动以**手掌触点为末端**，用 **3D 两骨 IK 反算肘、肩位置**，符合基本物理常识（无骨长突变、无反关节）。
- 新增**耐力条**：攀爬（伸手/悬挂）过程中耐力逐渐下降；抓住铆钉后进入快速恢复；耐力归零触发**摔落一段距离**，摔落通过**布娃娃系统**（Ragdoll Animator 2 物理混合）驱动，落定后可恢复到可攀爬状态。
- 角色升级为 **3D 可布娃娃驱动的骨骼模型**，接入 Ragdoll Animator 2；攀爬时动画/IK 驱动，摔落时切换全物理布娃娃。
- 新增 **3C 相机**：跟随角色并以合适角度展现墙面凸起与攀爬动作。
- 所有玩法参数（输入区、震动分档、IK 骨长与弯向、放大镜、起攀点、换手、耐力升降与摔落、相机、布娃娃混合等）统一由 **ScriptableObject** 配置，支持运行时实时调试。
- 非目标：不做联网/多人（沿用现有 `coop-networking`）；不做关卡编辑器、计分/成长、完整音频系统；不实现除攀爬与摔落外的移动方式；不移除旧的 2D 方向式原型，仅不作为主玩法入口。

## Capabilities

### New Capabilities

- `climb-character-3c`：定义 3D 手机攀爬游戏的角色攀爬 3C，包括左右手交替攀爬状态机、屏幕左右输入区与 3D 射线触点映射、RenderTexture 放大镜、墙面凸起铆钉抓力点与靠近度震动、手掌触点驱动的 3D 两骨 IK（肘/肩反算）、躯干跟随双手中点、耐力条与耗尽后的布娃娃摔落（Ragdoll Animator 2），以及基于 ScriptableObject 的可实时调参配置。

### Modified Capabilities

- 无。

## Impact

- 影响 `Assets/ClimbGame/`：新增 3D 攀爬 3C 模块（输入区、攀爬状态机、3D 手部 IK、铆钉、震动、RT 放大镜、耐力系统、布娃娃摔落接入、相机、配置 SO、组合根、演示场景）。
- 依赖并接入 `Assets/FImpossible Creations / Ragdoll Animator 2`（`RagdollAnimator2` / `RagdollHandler`，物理混合 Blend、冲击、起身）。
- 新增 3D 角色与场景资产（可布娃娃驱动骨骼模型、带凸起的墙体、立体铆钉、放大镜与输入区 UI），保持移动端简洁风格。
- 新增触觉反馈抽象层：Android 与 iOS **都是重点平台**，变强度震动需分别用 Android 振动器 amplitude 与 iOS CoreHaptics，能力不足时用脉冲频率调制回退；编辑器用可视化回退。
- 影响 `openspec/changes/add-climb-character-3c/` 与后续 `openspec/specs/climb-character-3c/`。
- 约束：Tuanjie Editor `1.9.3` / Unity `2022.3.62t11`，避免 Unity 6 专用 API。

## 验收标准

- 角色从墙面中心、双手默认位起攀；玩家先触发的一侧成为首个攀爬手，随后左右手交替连续完成至少 3 次攀爬，全程无报错。
- 3D 墙面凸起可见；铆钉为立体抓点；触点经射线映射到墙面，跟随、放大镜、靠近震动分档、抓到即换手、松手即放弃归位均可复现。
- 手跟随触点时肘/肩由 3D IK 反算，无骨长突变或反关节；躯干重心跟随双手中点移动。
- 耐力条随攀爬下降、抓住铆钉后快速恢复；耐力归零时角色通过布娃娃系统摔落一段距离并可恢复。
- Android 与 iOS 上震动分档可感知；关键参数改动可在运行时通过 ScriptableObject 生效（或近似实时）。
