## Context

本变更搭建 3D 手机攀爬游戏的核心攀爬 3C。玩家看到的是一面正对镜头、带**凸起/浮雕**的立体墙面，角色贴墙从中心起攀，用**左右手交替**攀爬：每次"攀爬"是当前手离开抓点、跟随触点寻找并抓住墙上一个**立体铆钉**的过程。角色接入布娃娃系统，平时由动画/IK 驱动，**耐力**耗尽时切换全物理布娃娃**摔落**一段距离。

技术约束：Tuanjie Editor `1.9.3`（底层 Unity `2022.3.62t11`），不使用 Unity 6 专用 API。沿用 `Assets/ClimbGame` 的分层理念（输入 / 角色 / 美术 / 组合根 / Editor），但旧的 2D 方向摇杆式 `ClimberController` 与本模型不同，本变更新增独立 3D 模块，不复用其移动逻辑。`Assets/DesignerSpace/MouseFollowJitter` 的"最近锚点→分档抖动"思路迁移为铆钉靠近度震动。布娃娃复用项目已内置的 `FImpossible Creations / Ragdoll Animator 2`（`RagdollAnimator2` + `RagdollHandler`，支持动画↔物理 Blend、冲击、起身）。

本轮已确定的关键取舍（回应上轮开放问题）：Android 与 iOS 同等重要；躯干重心跟随双手中点；起攀时双手都在默认位，首个攀爬手由玩家先触发的一侧决定；放大镜用 RenderTexture 实现。

## Goals / Non-Goals

**Goals:**

- 落地 3D 的"左右手交替 + 触点驱动 + 立体铆钉 + 手部 IK + 靠近度震动 + RT 放大镜 + 耐力 + 布娃娃摔落"完整攀爬循环。
- 手部运动符合物理常识：手掌触点为末端，3D 两骨 IK 反算肘、肩。
- 耐力驱动风险感：攀爬掉耐力、抓点回耐力、归零摔落。
- 所有关键参数由 ScriptableObject 配置，可运行时实时调试。
- 提供 3D 可布娃娃驱动角色、带凸起墙体、立体铆钉与演示场景，保持移动端简洁风格。
- 提供编辑器调试回退（鼠标模拟触点、震动可视化、耐力/状态调试显示），无真机也能验证。

**Non-Goals:**

- 不做联网、多人、关卡编辑器、计分/成长、完整音频系统。
- 不做除攀爬与摔落外的移动方式。
- 不追求真机变强度震动在所有机型精确一致，只保证分档可感知且可配置。
- 不移除旧的 2D 方向式原型；仅新增并作为新的主玩法入口。

## Decisions

### 1. 3D 场景与墙面凸起

- 墙面为 3D 网格，正对主相机；通过法线/高度或几何凸起体现**浮雕感**，铆钉是墙面上真实的立体凸起抓点（带碰撞体）。
- 攀爬平面近似为墙面，但触点映射用**射线与墙面碰撞**求交，从而落在凸起表面上，手可贴合凸起。
- 保持移动端性能：低多边形 + 法线贴图表现凸起优先于高模。

### 2. 屏幕左右输入区与 3D 触点映射（输入层）

- `ScreenTouchZones`：屏幕按配置分左右两个矩形输入区（可配分割比例、内边距、上下裁剪），支持触屏多点与编辑器鼠标回退。
- **输入门控**：只有与"当前攀爬手"同侧的输入区生效。
- **首手规则**：起攀时双手都在默认位，`current hand = none`；玩家在哪一侧输入区先触发，就以该侧手作为首个攀爬手，此后严格交替。
- **触点 → 墙面映射**：输入区内触点用 `Camera.ScreenPointToRay` 对墙面 collider 射线求交，得到 3D 目标点，作为当前手 IK 末端目标；支持灵敏度与偏移（手不必与手指重叠，避免遮挡）。
- 一次攀爬只跟踪一个 `fingerId`；抬起即结束本次攀爬（进入 `Grabbed` 或 `Abandoned`）。

### 3. 手部 3D 两骨 IK（肘/肩反算）

`ArmIkSolver`（每侧一份），链为 肩 → 上臂 → 肘 → 前臂 → 手掌，末端为手掌触点：

- 肩关节挂躯干上，随躯干受约束小幅移动（`shoulderSwayRange`），符合"反算肩关节位置"。
- 肘用两骨解析 IK（余弦定理）在由肩、末端与 `elbowBendDir`（弯向提示，通常朝身体外/远离墙面）确定的平面内求解。
- 末端超出 `armLength` 时夹到最大可达半径（`reachClamp`），不拉断、不反关节；末端目标做平滑。
- 输出关节世界坐标驱动骨骼；攀爬状态下由 IK 驱动，`RagdollHandler` 物理混合 Blend 设为偏动画。
- 优先使用 Ragdoll Animator 2 的 IK/handle 能力驱动手臂；若其 2D/自定义链接入受限，则用自研解析 IK 计算关节位姿再写入骨骼。

### 4. 铆钉抓力点与靠近度触觉反馈

- `RivetPoint`：墙面立体抓点组件，含 `snapRadius`（吸附/完成半径）与 Gizmo 预览（沿用 `AnchorPoint` 思路）。
- `RivetField`：管理场景所有铆钉，提供"给定手末端求最近铆钉与距离"。
- 靠近度 → 强度：距离越近强度越高；`intenseRadius`/`slightRadius` 定义档位，`snapRadius` 内触发一次强脉冲并完成攀爬（迁移自 `MouseFollowJitter` 三档，改为连续 + 分档，且用墙面 3D 距离）。
- `IHapticFeedback` 抽象 + 平台实现（**Android 与 iOS 同等重要**）：
  - Android：振动器 amplitude（API 26+）做变强度。
  - iOS：CoreHaptics / `UIImpactFeedbackGenerator`。
  - 能力不足：脉冲频率调制（越近脉冲越密）近似"越强"。
  - 编辑器/PC：屏幕可视化回退（放大镜边框/调试显示反映档位）。
- 只对"当前攀爬手"计算靠近度与反馈。

### 5. RenderTexture 放大镜观察窗

- `HandMagnifier`：触点附近显示小圆形放大区，放大当前手周围墙面凸起与着力点。
- 实现：一个小正交/透视副相机对准手部区域，渲染到 `RenderTexture`，用圆形遮罩显示在触点附近的 UI/叠层上。
- 可配置：倍率、圆半径、与触点偏移（避免被手指盖住）、边框随震动档位变色；伸手时显示，完成/松手后收起。

### 6. 攀爬状态机、耐力与躯干重心（角色层）

`ClimbHandStateMachine`：

- `Idle`：双手停在抓点/默认位，等待生效侧输入区触点。
- `Reaching`：检测到生效侧按下，进入伸手，触点映射为末端目标，IK 每帧更新；此阶段**耐力持续下降**。
- `Grabbed`：末端进入某铆钉 `snapRadius` → 强震动、吸附铆钉、判定本次攀爬完成 → **换手**、躯干重心朝**双手中点**移动、**耐力进入快速恢复**。
- `Abandoned`：抓握前触点抬起 → 当前手 `returnLerp` 回默认/上一抓点，回 `Idle`，**不换手**（此过程耐力仍视为消耗中，直到重新抓稳）。

耐力系统 `ClimbStamina`：

- 攀爬/悬挂（未稳定抓在铆钉上）时按 `drainRate` 下降；处于稳定抓握（`Grabbed` 后未开始新一次伸手）时按 `recoverRate` 快速恢复。
- 耐力归零 → 进入 `Falling`：解除 IK 抓握约束，`RagdollHandler` 物理混合 Blend 拉满切**全布娃娃**，角色受重力沿墙下落 `fallDistance`（或落到下方稳定点/地面），可加初速与冲击。
- 落定后按配置恢复到可攀爬状态（GetUp / 重置到最近有效抓点，重置耐力到 `recoverOnLandRatio`）。

躯干重心：非物理阶段，躯干目标位 = 两只手当前世界位的中点（可配 `centerOffset`/平滑），肩随之带动。

### 7. 布娃娃接入（Ragdoll Animator 2）

- 角色骨骼挂 `RagdollAnimator2`，攀爬时 `RagdollHandler` 混合偏动画（IK 主导），保留手部对铆钉的约束。
- 摔落时把物理混合拉满进入全布娃娃，利用其重力/冲击；可用 Impacts/Anchor 工具处理与墙、铆钉、地面的碰撞与吊挂。
- 落定恢复用其 GetUp/Blend 回到受控攀爬姿态。
- 全部布娃娃参数（混合曲线、摔落冲击、恢复时间）经配置暴露。

### 8. 3C 相机

- `ClimbCamera`：跟随躯干重心，带可配的偏移、俯仰角与阻尼，以合适角度展现墙面凸起与攀爬；摔落时可拉远/跟随下落。参数配置化。

### 9. 配置体系：ScriptableObject

拆分为若干 SO（组合根引用），字段带简体中文 Tooltip：

- `ClimbTuningConfig`：起攀点、首手规则、伸手/回收 lerp、触点灵敏度与偏移、输入区分割/内边距、躯干重心偏移与平滑。
- `ArmRigConfig`：上臂/前臂骨长、肩摆范围、肘弯朝向、可达半径夹取。
- `HapticConfig`：intenseRadius/slightRadius/snapRadius、各档强度/频率、抓握强脉冲、平台策略。
- `MagnifierConfig`：倍率、半径、偏移、边框反馈、RT 分辨率。
- `StaminaConfig`：drainRate、recoverRate、耗尽阈值、fallDistance、落地恢复比例。
- `RagdollFallConfig`：物理混合曲线、摔落冲击/初速、恢复时间、GetUp 策略。
- `ClimbCameraConfig`：偏移、俯仰、阻尼、摔落跟随。

组合根 `ClimbBootstrap` 运行时读取这些 SO；区分"即时生效"（如灵敏度、震动半径、耐力速率）与"需重新应用"（如放大镜相机、布娃娃结构）两类参数，后者提供 Inspector 重建按钮或 `OnValidate` 重应用。

## Risks / Trade-offs

- **变强度震动平台差异（Android+iOS 双端）**：`Handheld.Vibrate` 只有开/关。缓解：抽象 `IHapticFeedback`，分别用 Android amplitude / iOS CoreHaptics，不足退化脉冲频率调制，编辑器可视化。
- **Ragdoll Animator 2 与攀爬 IK 的耦合复杂度**：动画↔物理混合、手部约束与全布娃娃切换易出问题。缓解：分阶段先跑通纯 IK 攀爬，再单独调通"耗尽→全布娃娃→恢复"，用 Blend 平滑过渡；参数全部配置化以便调试。
- **触点被手指遮挡**：缓解：触点→手位置偏移 + RT 放大镜显示实际着力点。
- **IK 观感异常（反关节/抖动）**：缓解：固定 `elbowBendDir`、`reachClamp`、末端平滑、Gizmo 调试。
- **移动端性能（3D + RT 放大镜 + 布娃娃）**：缓解：低多边形 + 法线贴图表现凸起；放大镜 RT 用小分辨率、仅伸手时渲染；布娃娃仅摔落时全量物理。
- **耐力手感平衡**：drain/recover/fall 数值敏感。缓解：全部 SO 化，运行时调；提供调试显示。
- **与旧 ClimbGame 原型并存**：缓解：新增独立命名/目录，旧原型不接入新组合根。

## Migration Plan

1. 落地配置 SO 定义与数据结构（无行为），可在 Inspector 编辑。
2. 搭 3D 场景：带凸起墙体、立体铆钉、主相机与 3C 相机。
3. 输入区 + 3D 射线触点映射（鼠标回退优先），可视化触点。
4. 3D 两骨 IK 驱动手臂（先占位骨骼验证关节）。
5. 铆钉 + 靠近度检测 + 震动抽象（编辑器可视化回退）。
6. RT 放大镜观察窗。
7. 串左右手交替状态机 + 躯干双手中点重心 + 首手规则 + 组合根。
8. 接入耐力系统（下降/恢复/归零）。
9. 接入 Ragdoll Animator 2：攀爬混合偏动画，耗尽切全布娃娃摔落并恢复。
10. 生成 3D 角色/墙体/铆钉资产，替换占位。
11. 手工验收连续攀爬 + 摔落恢复，按 SO 调参收敛手感。

## Open Questions（已定案）

- 机型适配：**不做过度细化**，遇到具体机型问题再解决；震动抽象层保证可回退。
- 摔落落点规则：**固定下落 `fallDistance` 后吸附到最近铆钉**（行为可控），保留配置项以便日后切换自由物理落地。
- 耐力 UI：**做成屏幕条**（uGUI），显隐与样式可配置。
- 角色美术：原型阶段用**程序化基础体（低多边形/基本体拼装）**驱动，贴合移动端与凸起表现，后续可替换为正式模型。
