## ADDED Requirements

### Requirement: 关卡动线编辑器职责边界

关卡动线编辑器 MUST 作为编辑器内工具，允许策划在 Scene 视图墙面上打点并连接点之间的先后关系。工具 MUST NOT 在游戏运行时（Play/打包后）打点，MUST NOT 实现动线的运行时寻路或播放逻辑。

#### Scenario: 编辑器内使用

- **WHEN** 策划从菜单打开关卡动线编辑器窗口并进入打点模式
- **THEN** 工具 MUST 只在编辑器 Scene 视图内响应打点与连线操作，不影响运行时行为

### Requirement: 打点模式与墙面约束

工具 MUST 提供打点模式开关。进入打点模式后，策划在 Scene 视图点击时，工具 MUST 用射线检测命中的碰撞体，并且仅当命中碰撞体能取到 `Wall` 组件时才生成动线节点；否则 MUST 忽略该次点击。

#### Scenario: 在带 Wall 脚本的墙面打点

- **WHEN** 打点模式下策划点击一个带 `Wall` 脚本的碰撞体
- **THEN** 工具 MUST 在命中点（沿墙面法线做微小偏移）生成一个动线节点

#### Scenario: 点击非墙面被忽略

- **WHEN** 打点模式下策划点击一个不带 `Wall` 脚本的碰撞体（如地面、人偶）或空白处
- **THEN** 工具 MUST NOT 生成节点，并 SHOULD 给出"当前不是可打点墙面"的视觉提示

### Requirement: 有向先后关系与连线模式

工具 MUST 用有向边表达点之间的先后关系：先打的点指向后打的点。打点模式下连续打点 MUST 自动在上一个点与新点之间建立"上一个 -> 新点"的有向边。工具 MUST 另提供连线模式，允许策划依次点选两个已有节点，建立"第一个 -> 第二个"的有向边，支持一个点连接多个后继（自由有向图）。

#### Scenario: 连续打点自动串联

- **WHEN** 打点模式下策划先后打出点 A、点 B
- **THEN** 工具 MUST 建立一条从 A 指向 B 的有向边

#### Scenario: 手动连接任意两点

- **WHEN** 连线模式下策划先点选节点 A、再点选节点 B
- **THEN** 工具 MUST 建立一条从 A 指向 B 的有向边，且当该边已存在或 A 与 B 相同时 MUST NOT 重复建立

### Requirement: 虚线箭头可视化

工具 MUST 在 Scene 视图用虚线箭头绘制每条有向边，箭头方向指向后继节点，使先后关系可视。可视化 MUST 仅存在于编辑器，MUST NOT 生成运行时渲染对象。

#### Scenario: 边显示为指向后继的虚线箭头

- **WHEN** 两个节点之间存在一条从 A 指向 B 的有向边
- **THEN** Scene 视图 MUST 显示一条从 A 到 B 的虚线，并在靠近 B 处显示指向 B 的箭头

### Requirement: 数据持久化与可撤销

动线数据 MUST 以场景 GameObject 形式持久化并随场景保存，节点统一挂在自动创建的 `RouteNetwork` 层级下。打点与连线操作 MUST 接入 Undo，并标记场景为已修改。

#### Scenario: 撤销打点

- **WHEN** 策划打点或连线后按下撤销（Ctrl+Z）
- **THEN** 对应的节点或有向边 MUST 被回退

#### Scenario: 保存后动线仍在

- **WHEN** 策划保存场景后重新打开该场景
- **THEN** 之前生成的动线节点和有向边 MUST 仍然存在并正确显示
