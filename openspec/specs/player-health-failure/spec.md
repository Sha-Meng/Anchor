# player-health-failure Specification

## Purpose
定义主玩法玩家生命值、坠落伤害应用、生命归零失败状态和 MVP 失败弹窗反馈。

## Requirements
### Requirement: 玩家生命值状态
主玩法中每名本地权威玩家 MUST 拥有可配置最大生命值和当前生命值。当前生命值 MUST 在关卡开始或玩家状态初始化时设置为最大生命值，且 MUST 支持被 UI 或调试输出读取。

#### Scenario: 初始化生命值
- **WHEN** 主玩法初始化本地权威玩家
- **THEN** 系统 MUST 根据配置设置最大生命值，并将当前生命值初始化为最大生命值

#### Scenario: 读取生命值状态
- **WHEN** UI 或调试工具请求玩家生命状态
- **THEN** 系统 MUST 返回当前生命值、最大生命值和是否已经失败

### Requirement: 坠落伤害应用
生命系统 MUST 消费铆钉和绳索系统输出的结构化坠落伤害结果，并从对应坠落者的当前生命值中扣除建议伤害值。扣除后的当前生命值 MUST 不低于 0。生命系统 MUST NOT 重新计算坠落保护、绳索路径或第一个保护铆钉。

#### Scenario: 坠落后扣除生命
- **WHEN** 生命系统收到某名玩家的坠落伤害结果，且该玩家尚未失败
- **THEN** 系统 MUST 按伤害结果中的建议伤害值扣除该玩家当前生命值

#### Scenario: 生命值不低于零
- **WHEN** 坠落伤害值大于玩家当前生命值
- **THEN** 系统 MUST 将当前生命值设置为 0，而不是负数

#### Scenario: 不重新计算绳索伤害
- **WHEN** 生命系统应用坠落伤害结果
- **THEN** 系统 MUST 使用伤害结果中的建议伤害值和调试字段，并 MUST NOT 自行查询铆钉、绳索路径或玩家端点来重算伤害

### Requirement: 生命归零失败
当玩家当前生命值扣至 0 时，系统 MUST 立即进入失败状态。失败状态 MUST 阻止该玩家继续恢复攀爬、提交插锚、回收铆钉、收绳救援或常规攀爬输入。

#### Scenario: 生命归零触发失败
- **WHEN** 坠落伤害应用后玩家当前生命值等于 0
- **THEN** 系统 MUST 将该玩家标记为失败状态

#### Scenario: 失败后不恢复攀爬
- **WHEN** 玩家已经处于失败状态，且布娃娃坠落或受击恢复流程尝试恢复攀爬
- **THEN** 系统 MUST 阻止恢复到可攀爬状态

#### Scenario: 失败后禁用交互
- **WHEN** 玩家已经处于失败状态
- **THEN** 系统 MUST 拒绝该玩家继续提交攀爬输入、插锚、回收铆钉或收绳救援请求

### Requirement: 失败弹窗
主玩法 UI MUST 在玩家进入失败状态时显示 MVP 失败弹窗。失败弹窗 MUST 清楚表达游戏失败结果，并 MUST 在显示后保持可见，直到后续流程明确关闭或切换场景。

#### Scenario: 失败时显示弹窗
- **WHEN** 玩家生命值归零并进入失败状态
- **THEN** 主玩法 UI MUST 显示失败弹窗

#### Scenario: 弹窗保持可见
- **WHEN** 失败弹窗已经显示且当前没有重开、返回房间或切换场景流程
- **THEN** UI MUST 保持失败弹窗可见

#### Scenario: 非失败不显示弹窗
- **WHEN** 玩家受到坠落伤害但当前生命值仍大于 0
- **THEN** 主玩法 UI MUST NOT 显示失败弹窗
